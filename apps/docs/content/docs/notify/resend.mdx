## Overview

The Resend provider implements the [`email`](/docs/notify/channels/email) canal. Pass your Resend client instance — no raw HTTP calls.

## Install peer dependency

```bash
bun add resend
```

## Import

```ts
import { createResendCanal } from "@justwant/notify/channels/email/resend";
```

## Setup

```ts title="lib/notify.ts"
import { createNotify } from "@justwant/notify";
import { createResendCanal } from "@justwant/notify/channels/email/resend";
import { Resend } from "resend";

const emailCanal = createResendCanal({
  client: new Resend(process.env.RESEND_API_KEY!),
  from: "Acme <no-reply@acme.com>",
});

export const notify = createNotify({
  templates: [myTemplate],
  canals: { email: emailCanal },
});
```

## Options

### `ResendAdapterOptions`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `client` | `Resend` | Yes | Resend client instance (`new Resend(apiKey)`) |
| `from` | `string` | Yes | Default sender. Format: `"Name <email>"` or `"email"` |

## Environment variables

