## Overview

All adapters expose `adapter.transaction(fn)`. The callback receives a transactional adapter (`tx`) — use it exactly like the regular adapter. If `fn` throws, the transaction is rolled back automatically. If it returns successfully, the transaction is committed.

```ts
await adapter.transaction(async (tx) => {
  const txUsers = tx.defineTable(usersTable, userContract, mapping);
  const txPosts = tx.defineTable(postsTable, postContract, postMapping);

  const user = await txUsers.create({ email: "alice@example.com", name: "Alice" });
  await txPosts.create({ userId: user.id, title: "Hello world" });
});
```

If `txPosts.create` throws, `txUsers.create` is rolled back — no user is created.

## Drizzle

```ts title="lib/db.ts"
import { createDrizzleAdapter } from "@justwant/db/drizzle";
import { drizzle } from "drizzle-orm/bun-sqlite";
import { Database } from "bun:sqlite";

const sqlite = new Database("app.db");
export const adapter = createDrizzleAdapter(drizzle(sqlite), { dialect: "sqlite" });
```

```ts title="services/transfer.ts"
import { adapter } from "@/lib/db";
import { accounts } from "@/tables/accounts";

export async function transferBalance(fromId: string, toId: string, amount: number) {
  await adapter.transaction(async (tx) => {
    const txAccounts = tx.defineTable(accountsTable, accountContract, mapping);

    const from = await txAccounts.findById(fromId);
    const to = await txAccounts.findById(toId);

    if (!from || !to) throw new Error("Account not found");
    if (from.balance < amount) throw new Error("Insufficient balance");

    await txAccounts.update(fromId, { balance: from.balance - amount });
    await txAccounts.update(toId, { balance: to.balance + amount });
  });
}
```

## Prisma

Prisma transactions work the same way. The `tx` adapter wraps a Prisma interactive transaction client:

```ts
await adapter.transaction(async (tx) => {
  const txUsers = tx.defineTable("user", userContract, mapping);
  const txAudit = tx.defineTable("auditLog", auditContract, auditMapping);

  const user = await txUsers.update(userId, { name: newName });
  await txAudit.create({
    userId,
    action: "update",
    payload: JSON.stringify({ name: newName }),
  });
});
```

## Waddler

With Waddler, use `db` directly inside the transaction — `Db` also exposes `table()` and `defineTable()`:

```ts title="lib/db.ts"
import { createDb } from "@justwant/db/waddler";
import { createBunSqliteAdapter } from "@justwant/db/bun-sqlite";

export const db = createDb(createBunSqliteAdapter({ connection: "app.db" }));
```

```ts title="services/order.ts"
import { db } from "@/lib/db";

export async function placeOrder(userId: string, items: OrderItem[]) {
  await db.transaction(async (tx) => {
    const txOrders = tx.table(orderContract);
    const txItems = tx.table(orderItemContract);
    const txInventory = tx.table(inventoryContract);

    const order = await txOrders.create({ userId, status: "pending" });

    for (const item of items) {
      await txItems.create({ orderId: order.id, ...item });

      const stock = await txInventory.findOne({ productId: item.productId });
      if (!stock || stock.quantity < item.quantity) {
        throw new Error(`Insufficient stock for ${item.productId}`);
      }
      await txInventory.update(stock.id, { quantity: stock.quantity - item.quantity });
    }
  });
}
```

## Error handling and rollback

Throw any error inside the callback to trigger a rollback:

```ts
await adapter.transaction(async (tx) => {
  const txAccounts = tx.defineTable(accountsTable, accountContract, mapping);

  const account = await txAccounts.findById(id);
  if (!account) {
    throw new Error("Account not found"); // triggers rollback
  }

  await txAccounts.update(id, { balance: account.balance + 100 });
});
```

The transaction is atomic — either all operations succeed, or none do.

## Reading inside a transaction

Always use the transactional table (`tx.defineTable(...)`) for reads inside a transaction. Reading from the non-transactional table may return stale data or bypass isolation guarantees.

```ts
await adapter.transaction(async (tx) => {
  const txUsers = tx.defineTable(usersTable, userContract, mapping);

  // ✓ reads from the same transaction
  const user = await txUsers.findById(id);

  // ✗ reads outside the transaction — may see uncommitted state from other transactions
  // const user = await users.findById(id);

  await txUsers.update(id, { name: "Alice" });
});
```

## Nesting

Drizzle and Prisma support savepoints for nested transactions. With most SQLite drivers, nested transactions are not supported — avoid nesting or use a flat transaction with all operations at the same level.
