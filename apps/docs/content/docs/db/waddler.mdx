## Overview

The Waddler adapter provides raw SQL access via tagged template literals (`sql\`...\``). Unlike Drizzle or Prisma, Waddler requires no schema file or code generation step — it works directly with SQL.

The Waddler integration in `@justwant/db` adds:
- The same `MappedTable` interface (`create`, `findById`, `findMany`, etc.)
- Contract-based validation before insert and update
- DDL helpers (`createTable`, `exist`, `drop`)
- Soft delete support

## Define a table

```ts title="tables/users.ts"
import { db } from "@/lib/db";
import { defineContract, field } from "@justwant/contract";

const userContract = defineContract({
  id: field.string(),
  email: field.string(),
  name: field.string(),
  createdAt: field.string(),
  deletedAt: field.string().optional(),
});

const mapping = {
  id: { name: "id" },
  email: { name: "email" },
  name: { name: "name" },
  createdAt: { name: "created_at" },
  deletedAt: { name: "deleted_at" },
};

export const users = db.defineTable("users", userContract, mapping);
```

## DDL helpers

The Waddler adapter adds three DDL methods unavailable in Drizzle and Prisma adapters:

```ts
// Create table (IF NOT EXISTS)
await users.createTable();

// Check if table exists
const exists = await users.exist();

// Drop table
await users.drop();
```

## createSafe / updateSafe

The Waddler adapter also exposes validation-returning variants that avoid throwing:

```ts
const result = await users.createSafe({
  email: "alice@example.com",
  name: "Alice",
  createdAt: new Date().toISOString(),
});

if (!result.ok) {
  console.error(result.issues);
} else {
  console.log(result.value.id);
}
```

## Soft delete

Same as other adapters: `delete` sets `deletedAt`, `hardDelete` removes the row. Soft-deleted rows are excluded from all `find*` queries.

To change or disable the soft-delete column:

```ts
db.defineTable("users", userContract, mapping, {
  softDeleteColumn: "archivedAt", // custom column
});

db.defineTable("users", userContract, mapping, {
  softDeleteColumn: null, // disable
});
```

## Transactions

<Callout type="warn">
  The Waddler adapter does **not** implement `transaction()`. The method is declared as optional in the `Db` interface but is never assigned by `createWaddlerAdapter`, so calling `db.transaction()` will throw a `TypeError: db.transaction is not a function` at runtime.

  If you need transaction support, use the [Drizzle](/docs/db/drizzle) or [Prisma](/docs/db/prisma) adapters, both of which implement `transaction()`.
</Callout>
