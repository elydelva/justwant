## Overview

The `/password` module provides Argon2id password hashing and verification. Argon2id is the recommended algorithm from [OWASP](https://cheatsheetseries.owasp.org/cheatsheets/Password_Storage_Cheat_Sheet.html) for password storage — it is resistant to GPU and side-channel attacks.

Internally it uses the `argon2` npm package (native binding, audited) with fixed secure parameters:

| Parameter | Value |
|-----------|-------|
| Algorithm | Argon2id |
| Time cost | 3 iterations |
| Memory cost | 64 MiB |
| Parallelism | 1 thread |

## Import

```ts
import { hashPassword, verifyPassword } from "@justwant/crypto/password";
```

## API

### `hashPassword(password)`

Hashes a password with Argon2id. Returns a PHC-format string suitable for storage.

```ts
const hash = await hashPassword("hunter2");
// "$argon2id$v=19$m=65536,t=3,p=1$..."
```

### `verifyPassword(password, hash)`

Verifies a plain-text password against a stored Argon2id hash. Returns `true` if the password matches, `false` otherwise.

```ts
const ok = await verifyPassword("hunter2", storedHash);
```

## Signup and login pattern

```ts title="lib/auth.ts"
import { hashPassword, verifyPassword } from "@justwant/crypto/password";

// On signup: hash before storing
export async function createUser(email: string, password: string) {
  const hash = await hashPassword(password);
  await db.users.create({ email, passwordHash: hash });
}

// On login: verify against stored hash
export async function loginUser(email: string, password: string) {
  const user = await db.users.findByEmail(email);
  if (!user) return null;

  const ok = await verifyPassword(password, user.passwordHash);
  if (!ok) return null;

  return user;
}
```

**Important**: Never store plain-text passwords. Always store the hash returned by `hashPassword`, not the original password.

**Note**: `hashPassword` is CPU and memory intensive by design (64 MiB per operation). Run it outside the hot path and avoid running many concurrent hashes in a single process.
