## Overview

The `/rotating-encryption` module provides a **keyring** — a collection of AES-256-GCM keys that supports seamless key rotation. It lets you introduce a new encryption key without immediately re-encrypting all existing data: old ciphertexts continue to decrypt with the previous key while all new writes use the new primary key.

This is the right tool when you need to rotate encryption keys periodically (e.g., for compliance), or when you suspect a key may have been compromised and want to move to a new one without downtime.

## Import

```ts
import { createKeyring } from "@justwant/crypto/rotating-encryption";
// alias:
import { createRotatingEncryption } from "@justwant/crypto/rotating-encryption";
```

## Usage

### Basic encryption and decryption

```ts
import { createKeyring } from "@justwant/crypto/rotating-encryption";
import { deriveKey } from "@justwant/crypto/derive-key";

// Derive a 32-byte key from your master secret
const key = deriveKey(process.env.MASTER_SECRET!, "production", "encrypt/pii", 32);

const keyring = createKeyring([key]);

// Encrypt
const plaintext = new TextEncoder().encode("john.doe@example.com");
const ciphertext = keyring.encrypt(plaintext);

// Decrypt
const result = keyring.decrypt(ciphertext);
const original = new TextDecoder().decode(result!);
// "john.doe@example.com"
```

### Rotating to a new key

Call `rotate()` to prepend a freshly-generated random key to the keyring. From that point on, all new encryptions use the new key. Existing ciphertexts encrypted with the old key still decrypt successfully.

```ts
// Add a new primary key (old key is retained for decryption)
keyring.rotate();

// New data is encrypted with the new key
const newCiphertext = keyring.encrypt(plaintext);

// Old ciphertext is still decryptable
const old = keyring.decrypt(ciphertext);   // works
const newer = keyring.decrypt(newCiphertext); // works
```

Once all data has been re-encrypted with the new key, you can drop the old key from the keyring by creating a fresh one with only the new key.

### Multiple keys at initialization

You can also pass multiple keys at creation time (e.g., when loading previously rotated keys from a secrets store):

```ts
import { createKeyring } from "@justwant/crypto/rotating-encryption";
import { deriveKey } from "@justwant/crypto/derive-key";

const primary = deriveKey(process.env.MASTER_SECRET!, "prod", "encrypt/pii/v2", 32);
const legacy  = deriveKey(process.env.MASTER_SECRET!, "prod", "encrypt/pii/v1", 32);

// primary is used for all new encrypts; legacy is tried on decrypt if primary fails
const keyring = createKeyring([primary, legacy]);
```

## API Reference

### `createKeyring(keys)`

Creates a `Keyring` instance.

| Parameter | Type | Description |
|-----------|------|-------------|
| `keys` | `Uint8Array[]` | Array of 32-byte AES-256-GCM keys. Must contain at least one key. The first key is the primary (used for encryption). |

Throws if `keys` is empty or any key is not exactly 32 bytes.

Also exported as `createRotatingEncryption` (alias).

---

### `Keyring` interface

#### `encrypt(plaintext)`

Encrypts a `Uint8Array` with the primary (first) key using AES-256-GCM. Returns a `Uint8Array` containing a key-index byte, a 12-byte nonce, and the ciphertext + GCM authentication tag.

| Parameter | Type | Description |
|-----------|------|-------------|
| `plaintext` | `Uint8Array` | Raw bytes to encrypt |

Returns `Uint8Array`.

#### `decrypt(ciphertext)`

Decrypts a `Uint8Array` produced by `encrypt()`. Reads the key-index byte from the ciphertext and tries that key first, then falls back to subsequent keys in the keyring if needed.

| Parameter | Type | Description |
|-----------|------|-------------|
| `ciphertext` | `Uint8Array` | Output of `encrypt()` |

Returns `Uint8Array | null`. Returns `null` if the ciphertext is malformed or no key in the keyring can decrypt it.

#### `rotate()`

Generates a new random 32-byte key and prepends it to the keyring, making it the new primary. Takes no arguments and returns `void`.

All subsequent calls to `encrypt()` use this new key. Existing ciphertexts remain decryptable with the previous key(s).

## Key rotation workflow

1. Call `keyring.rotate()` to introduce a new primary key.
2. Continue serving traffic — old ciphertexts still decrypt, new writes use the new key.
3. In a background job, re-encrypt old records: `encrypt(decrypt(old))`.
4. Once all records are re-encrypted, create a new keyring with only the current primary key to drop the legacy key from memory.
