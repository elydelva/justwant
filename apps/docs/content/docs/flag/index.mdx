## What is @justwant/flag?

`@justwant/flag` is a **type-safe feature flag engine** for TypeScript applications. It lets you define flags with evaluation rules, gradually roll out features, and remotely override rule configuration — all without vendor lock-in.

**Use it when** you need to:
- Enable features for specific users, orgs, or contexts
- Gradually roll out to a percentage of users
- Override flag behavior at runtime without redeploying
- Define time-based diffusion curves (Bézier, stepped)

## Features

| Feature | Description |
|---------|-------------|
| **Typed rules** | `defineRule` with config schema, context schema, and logic function |
| **Typed flags** | `defineFlag` attaches rules to a feature and defines evaluation strategy |
| **Config overrides** | Remote config stored in a repo, updateable at runtime |
| **Rollout** | `rollout(hash, threshold)` — deterministic percentage rollout |
| **Diffusion** | `diffusionBezier` — smooth time-based rollout via Bézier curves |
| **Diffusion** | `diffusionStepped` — time-based rollout via discrete percentage steps |
| **Memory repo** | `createMemoryFlagConfigRepo` for tests and development |

## How it works

1. **Define rules** — Each rule has a logic function, optional config schema (remote), and optional context schema (request data).
2. **Define flags** — Attach rules to a feature. Set `strategy: "any"` (default) or `"all"`.
3. **Create a service** — Inject a flag config repo and your flag definitions.
4. **Evaluate** — Call `service.evaluate(flag, context)` to get a boolean.
