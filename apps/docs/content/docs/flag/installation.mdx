## Install

```bash
bun add @justwant/flag
```

## Quick setup

The three steps to get a flag running are: define a rule, define a flag, create a service.

```ts title="lib/flags.ts"
import { defineRule, defineFlag, createFlagService, createMemoryFlagConfigRepo } from "@justwant/flag";
import { defineFeature } from "@justwant/feature";
import * as v from "valibot";

// 1. Define a rule
const betaRule = defineRule({
  name: "beta-access",
  config: v.object({ enabled: v.boolean() }),
  context: v.object({ userId: v.string() }),
  defaultConfig: { enabled: false },
  logic: ({ config }) => config.enabled,
});

// 2. Define a flag
const betaFeature = defineFeature({ name: "beta" });
const betaFlag = defineFlag(betaFeature, {
  default: false,
  rules: [betaRule],
});

// 3. Create a service
const repo = createMemoryFlagConfigRepo(); // swap for a DB-backed repo in production
export const flagService = createFlagService({ flags: [betaFlag], repo });
```

## Evaluate a flag

```ts title="app/route.ts"
import { flagService, betaFlag } from "@/lib/flags";

const enabled = await flagService.evaluate(betaFlag, { userId: "user_123" });

if (enabled) {
  // show beta feature
}
```

## Dev vs. production repos

`createMemoryFlagConfigRepo` stores overrides in memory — ideal for tests and local development. In production, replace it with a database-backed implementation that satisfies the `FlagConfigRepo` interface.

**Note**: See [Config Overrides](/docs/flag/config-overrides) for the full `FlagConfigRepo` interface and remote config patterns.
